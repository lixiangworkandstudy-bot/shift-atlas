# Echo Portfolio - 开发规则

## Git 分支管理
- **禁止在 main 分支直接修改代码**
- 所有功能开发必须在 `feat/*` 分支进行
- 示例：`feat/add-contact-form`、`feat/update-hero-section`

## 代码修改规范
- **一次只改一个功能点** - 保持改动聚焦和可追踪
- **修改范围限制**：
  - 主要修改应在 `src/` 目录内
  - 优先修改 `src/app/` 和 `src/components/`
  - 避免修改配置文件，除非功能明确需要

## 质量保证
- **每次改动后必须验证 `npm run dev` 可正常运行**
- 提交前确保没有 TypeScript 错误
- 保持现有功能不被破坏

## 依赖管理
- **不要随意升级依赖版本**
- **不要新增大型库**，除非：
  1. 明确说明添加原因
  2. 评估包大小影响
  3. 考虑是否有更轻量替代方案
- 优先使用项目已有的依赖

## 代码风格
- 遵循项目现有的代码风格
- 保持像素艺术和复古未来主义的设计语言
- 组件命名遵循现有模式
